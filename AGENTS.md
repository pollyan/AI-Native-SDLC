# AGENTS.md — AI-Native SDLC Agent 操作指南

> 适用于所有在本仓库工作的 AI Agent。

---

## 这个仓库是什么

本仓库是 **AI-Native SDLC 框架**的设计资产库——一套规范驱动的开发方法论，**不是可运行的应用程序**。

在处理任何任务前，必须先读 `CONTEXT.md`（项目领域词汇表），再读 `specs/` 下的主框架文档。

---

## 目录结构

| 路径 | 职责 |
|------|------|
| `CONTEXT.md` | 项目领域词汇表，所有 Agent 必读，是术语的唯一真相来源 |
| `AGENTS.md` | 本文件，AI Agent 行为规范 |
| `specs/` | 框架的"需求侧"——框架整体设计文档、各阶段的设计讨论与决策记录 |
| `skills/` | 框架的"交付侧"——可直接使用的框架产物（Skill、模板等），每个阶段自包含 |
| `archive/` | 历史版本与设计决策记录，不参与当前工作，仅供回溯 |
| `references/` | 竞品参考资料，已加入 `.gitignore`，不进入版本控制 |

---

## 核心原则

**1. 人主控，AI 辅助**
AI 可以建议，但不自动执行阶段切换。所有关键决策由 Developer 确认。

**2. Spec 文件是阶段间唯一传递载体**
不通过口头描述传递阶段信息，必须写入 Spec 文件。

**3. 统一语言是基础设施**
发现术语冲突时立即拦截；新术语确定时立即写入 `CONTEXT.md`，不批量处理。

**4. 垂直切片驱动执行**
每个切片贯穿所有集成层，禁止水平切片（先写完所有 schema → 再写所有 API）的反模式。

---

## 工作流

```
Requirement Review → Plan → Execute → Review
```

每个阶段产出一份 Spec 文件，作为下一阶段的输入。详见主框架文档。

---

## AI 行为边界

- 使用 `CONTEXT.md` 中的术语，不猜测、不自行定义
- 每个阶段完成后必须执行自审核检查，不跳过门控
- 不擅自修改切片范围；Execute 阶段发现问题时反馈给 Developer，由其决定是否回退
- 不确定的内容用 `[NEEDS CLARIFICATION]` 标记
- Developer 要求执行就执行到底，不反复询问"是否继续"
- 调试时生成 3-5 个可证伪假设，不锚定第一个猜测

---

## 文档修改规范

- **主框架文档**（`docs/` 下）是单一真相来源（SSOT），修改前检查版本历史
- **`CONTEXT.md`** 实时维护，格式遵守现有结构
- **重大设计决策**在 `archive/` 下的决策记录文件中留痕
- `references/` 已加入 `.gitignore`，不进入版本控制
